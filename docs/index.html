Quick Start

Five minutes
to your first test.

Register actions on a client. They chain as a linked list — when action N finishes for a virtual user, action N+1 starts. Configure the environment and run.

Sustained

Maintains N concurrent users continuously — each user loops through the action chain until the test ends.

Chained

Actions execute in sequence. Session state flows between steps — login, browse, checkout.

LoadTest.java 
var client = DefaultLoadTestClient.blocking();

// Actions chain as a linked list
client.execute("login", session -> {
    var resp = httpClient.send(loginReq, handler);
    session.put("token", resp.headers()
        .firstValue("Authorization").orElseThrow());
});

client.execute("browse", session -> {
    String token = (String) session
        .get("token").orElseThrow();
    // use token from login step
});

var config = EnvironmentConfig.create()
    .numberOfUsers(100)
    .rampUpTime(Duration.ofSeconds(30))
    .testDuration(Duration.ofMinutes(5))
    .connectionPoolSize(100);

var env = new LocalEnvironment(config);
LoadTestRun run = env.start(client);

// Wait for completion
TestResult result = run.result().get();
Internals

One pool per action.

Each action in the chain gets its own connection pool. Users move forward from pool to pool, freeing space for the next round. Pool metrics power the dashboard — one line per action.

Features

Built for real
load testing.

Everything you need to stress-test your services, with a clean API that stays out of your way.

⚡ Virtual Threads Java 21

Blocking clients automatically use virtual threads. Handle thousands of concurrent users without thread pool tuning. Override per-environment.

📈 Micrometer Metrics Default

Response times, percentiles, success/failure counters, active users — all collected via Micrometer. Plug in Prometheus, Datadog, or any registry.

🔨 OpenTelemetry Ready Pluggable

Implement the MetricsCollector interface with your OTel SDK setup. The API is designed for drop-in replacement.

🌐 Web Dashboard Real-time

Embedded Tomcat serves a live dashboard with Chart.js graphs. SSE endpoint streams metrics every second. Start and stop tests from the UI.

🔗 Action Chains

Register multiple actions that execute as a linked list. Session state passes between steps. Each action gets its own pool and metrics line.

📈 JSON Log Output

Every execution record is written to a JSONL stream file. Final results include per-action summaries and time-series snapshots for charting.

🚀 Ramp-up Control

Linear ramp-up from zero to peak users. Configure the duration independently from the test length. Smooth user addition every 200ms.

🔧 Extensible Clients

ClientFactory SPI for registering custom client types. Build HTTP, gRPC, JDBC, or WebSocket clients — the library doesn't care what you test.
